LV Interface for Amazon S3.lvlib:Upload Part.vi

This operation uploads a part in a multipart upload. You must initiate a multipart upload (see Initiate Multipart Upload.vi) before you can upload any part. In response to your initiate request, you will receive a Multipart Upload Refnum, that you must use through the whole upload process.

Multipart Upload RefNum in

Multipart Upload RefNum in identifies the multipart upload whose part is going to be uploaded. Use the Initiate Multipart Upload.vi to obtain a RefNum.

Part Number

Part Number can be any number from 1 to 10,000, inclusive. A part number uniquely identifies a part and also defines its position within the object being created.

If you upload a new part using the same part number that was used with a previous part, the previously uploaded part is overwritten.

The first part of the object has to be Part Number 1.

Default value: 1

error in (no error)

error in can accept error information wired from VIs previously called. Use this information to decide if any functionality should be bypassed in the event of errors from other VIs.

Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    status

    status is TRUE (X) if an error occurred or FALSE (checkmark) to indicate a warning or that no error occurred.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    code

    code is the error or warning code.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    source

    source describes the origin of the error or warning.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

Data

Data is the RAW data of the current part of the object.

Note: Each part must be at least 5 MB in size, except the last part.There is no size limit on the last part of your multipart upload.

Content MD5

Content MD5 is the base64 encoded 128-bit MD5 digest of the Data according to RFC 1864. This information can be used as a message integrity check to verify that the data is the same data that was originally sent. Although it is optional, Amazon recommends using the Content-MD5 mechanism as an end-to-end integrity check.

Default value: empty string - MD5 is not used

Region

Region specifies the geographical location of the bucket.

Multipart Upload RefNum out

Multipart Upload RefNum out is the same RefNum as Multipart Upload RefNum in. You have to use this RefNum with other Multipart Upload VIs during the upload process.

Success?

Success? is TRUE if the upload of the part completed successfully, is FALSE otherwise.
