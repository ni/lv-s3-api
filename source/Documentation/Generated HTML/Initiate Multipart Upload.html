LV Interface for Amazon S3.lvlib:Initiate Multipart Upload.vi

This operation initiates a multipart upload operation and returns a Multipart Upload Refnum. After initiating a multipart upload process, you can start uploading the individual parts of the object. Multipart uploads provide the following advantages: - Improved throughput: You can upload parts in parallel to improve throughput. - Quick recovery from any network issues: Smaller part size minimizes the impact of restarting a failed upload due to a network error. - Pause and resume object uploads: You can upload object parts over time. Once you initiate a multipart upload there is no expiry; you must explicitly complete or abort the multipart upload. - Begin an upload before you know the final object size: You can upload an object as you are creating it.

Request Parameters

Request Parameters allows to specify some advanced features of S3 to use to store the object.

    Content Type

    Content Type is the standard MIME type describing the format of the contents. For more information, go to http://www.w3.org/Protocols/rfc2616/rfc2616-sec14.html#sec14.17.

    Default value: binary/octet-stream

    ACL

    ACL is the canned ACL to apply to the object. You can limit access of other people to the object by setting it's ACL.

    Default value: private - Only it's owner can access the object

    Storage Class

    Storage Class allows you to specify the type of storage in terms of redundancy that will be used to store data.

    Possible values are:

    - STANDARD is the default, highly redundant storage type Amazon uses.

    - REDUCED_REDUNDANCY enables customers to reduce their costs by storing non-critical, reproducible data at lower levels of redundancy than Amazon S3's standard storage.

    User Metadata

    User Metadata array allows you to store some metadata associated with the object.

      Object Metadata

      Object Metadata represents a metadata associated with an object in the form of a key-value pair.

        Key

        Key specifies the name of the metadata you want to store with the object.

        Value

        Value specifies the value of the actual metadata referenced by Key you want to store with the object.

Object Path

Object Path specifies the relative path from the bucket's root to the object you want to store.

Note: leading '/' is not necessary, but will still work.

Bucket

Bucket specifies the name of the bucket you want to use to store the object.

Region

Region specifies the geographical location of the bucket.

Multipart Upload RefNum

Multipart Upload RefNum is a unique RefNum that identifies a specific Multipart Upload. Use this RefNum with the Multipart Upload VIs throughout the whole upload process.
