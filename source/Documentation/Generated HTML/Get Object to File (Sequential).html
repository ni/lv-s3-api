LV Interface for Amazon S3.lvlib:Get Object to File (Sequential).vi

This operation retrieves an object from Amazon S3 to a file. The operation retrieves object data in chunks. You must have READ access to the object. If you grant READ access to the anonymous user, you can return the object without using an authorization header. To distribute large files to many people, you can save bandwidth costs using BitTorrent.

Chunk size (kB)

Chunk size (kB) specifies the chunk size in kBytes to use while downloading the object.

This download operation is sequential, so the object will be downloaded in chunks, one part after the other.

You can use this input to limit the memory usage of the operation.

Default value: 1024 kB (1 MB)

Object Path

Object Path is the relative path from the bucket's root to the object you want to get size information of.

Note: leading '/' is not necessary, but will still work.

Bucket

Bucket specifies the name of the bucket where the object you want to download resides.

File path

File path is the absolute path on the local filesystem, where the object will be downloaded.

Version ID

Version ID may be used, if Versioning is enabled on the bucket, and you want to download a specific version of the object. If no Version ID is specified, the latest version of the object will be downloaded.

Default value: empty string - Use the latest version

Region

Region specifies the geographical location of the bucket.
