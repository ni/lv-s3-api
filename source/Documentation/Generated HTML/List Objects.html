LV Interface for Amazon S3.lvlib:List Objects.vi

This operation returns some or all (default) of the objects in a bucket. To use this operation, you must have READ access to the bucket.

error in (no error)

Max keys

Max keys may be used as a selection criteria to return a subset of all the objects in a bucket.

If you wire a number greater than 0, the list will contain the first "Max keys" number of objects of the bucket.

Default value: 0, which will return all elements.

Contents

Contents array lists information about the objects stored in the specified bucket.

    Content

    Content contains information about an object.

      Key

      Key is the key of the object. Key contains the full path of the object relative to the bucket root.

      Note: Key does not contain the leading "/" that represents the bucket root.

      LastModified

      LastModified shows the Last modification time (in UTC) of the object.

      ETag

      ETag is the Entity Tag (which is an MD5 Sum) of the object. ETag only reflects changes to the contents of an object, not its metadata.

      Size

      Size is the size of the object in bytes.

      Storage Class

      Storage Class specifies the type of storage in terms of redundancy that is used to store the object.

      Possible values are:

      - STANDARD is the default, highly redundant storage type Amazon uses.

      - REDUCED_REDUNDANCY enables customers to reduce their costs by storing non-critical, reproducible data at lower levels of redundancy than Amazon S3's standard storage.

      Owner

      Owner contains elements that identify the object owner.

        ID

        ID is the Canonical User ID of the object owner.

        DisplayName

        DisplayName is the screen name (Email address) of the object owner.
