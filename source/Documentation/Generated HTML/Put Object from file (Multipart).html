LV Interface for Amazon S3.lvlib:Put Object from file (Multipart).vi

This operation uploads an object from a file to a bucket in multiple parts. You can set Chunk size (kB) to limit memory usage of the operation.

Local file path

Local file path is the absolute path to the file on the local filesystem that you want to upload.

error in (no error)

error in can accept error information wired from VIs previously called. Use this information to decide if any functionality should be bypassed in the event of errors from other VIs.

Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    status

    status is TRUE (X) if an error occurred or FALSE (checkmark) to indicate a warning or that no error occurred.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    code

    code is the error or warning code.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    source

    source describes the origin of the error or warning.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

Request Parameters

Request Parameters allows to specify some advanced features of S3 to use to store the object.

    Content Type

    Content Type is the standard MIME type describing the format of the contents. For more information, go to http://www.w3.org/Protocols/rfc2616/rfc2616-sec14.html#sec14.17.

    Default value: binary/octet-stream

    ACL

    ACL is the canned ACL to apply to the object. You can limit access of other people to the object by setting it's ACL.

    Default value: private - Only it's owner can access the object

    Storage Class

    Storage Class allows you to specify the type of storage in terms of redundancy that will be used to store data.

    Possible values are:

    - STANDARD is the default, highly redundant storage type Amazon uses.

    - REDUCED_REDUNDANCY enables customers to reduce their costs by storing non-critical, reproducible data at lower levels of redundancy than Amazon S3's standard storage.

    User Metadata

    User Metadata array allows you to store some metadata associated with the object. The total size of the HTTP request that includes the User Metadata - not including the body ( Data)-, must be less than 8 KB.

      Object Metadata

      Object Metadata represents a metadata associated with an object in the form of a key-value pair.

        Key

        Key specifies the name of the metadata you want to store with the object.

        Value

        Value specifies the value of the actual metadata you want to store with the object.

Object Path

Object Path specifies the relative path from the bucket's root to the object you want to store.

Note: leading '/' is not necessary, but will still work.

Bucket

Bucket specifies the name of the bucket you want to use to store the object.

Chunk size (kB)

Chunk size (kB) specifies the chunk size in kBytes to use while uploading the object.

This upload operation is sequential, so the object will be uploaded in chunks, one part after the other.

You can use this input to limit the memory usage of the operation.

If Chunk size is larger than file size, the operation will be completed using only one part, and chunk size will have no effect.

Default value: 5120 kB (5 MB)

Region

Region specifies the geographical location of the bucket.

CompleteMultipartUploadResult

CompleteMultipartUploadResult contains information about the object if it has been successfully created.

    Location

    Location is the URI that identifies the newly created object.

    Bucket

    Bucket is the name of the bucket that contains the newly created object.

    Key

    Key is the object key that contains the full path of the object relative to the bucket root. Key does not contain the leading "/" that represents the bucket root.

    ETag

    ETag contains the Entity Tag (which is MD5 sum) of the stored object.

Version ID

Version ID contains the Version ID of the stored object if Versioning is enabled on the bucket.
