LV Interface for Amazon S3.lvlib:Get Object Size.vi

This operation returns Object size (in bytes). Note: To determine object size, the last byte of the object is retreived via a Get Object request.

Object Path

Object Path is the relative path from the bucket's root to the object you want to get size information of.

Note: leading '/' is not necessary, but will still work.

Bucket

Bucket specifies the name of the bucket where the object whose size you want to get resides.

error in (no error)

error in describes error conditions that occur before this node runs. This input provides standard error in functionality.

    status

    status is TRUE (X) if an error occurred before this node ran or FALSE (checkmark) to indicate a warning or that no error occurred before this node ran. The default is FALSE.

    code

    code is the error or warning code. The default is 0.

    source

    source specifies the origin of the error or warning and is, in most cases, the name of the node that produced the error or warning. The default is an empty string.

Version ID

Version ID may be used, if Versioning is enabled on the bucket, and you want to get the size of a specific version of the object. If no Version ID is specified, the latest version of the object will be used.

Default value: empty string - Use the latest version

Region

Region specifies the geographical location of the bucket.

Object Size (bytes)

Object Size (bytes) returns the size of the specified object in bytes.
