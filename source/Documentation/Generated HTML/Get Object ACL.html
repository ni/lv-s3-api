LV Interface for Amazon S3.lvlib:Get Object ACL.vi

This operation returns the access control list (ACL) of an object. To use this operation, you must have READ_ACP access to the object.

Bucket

Bucket specifies the name of the bucket where the object whose ACL information you want to get resides.

Object Path

Path is the relative path from the bucket's root to the object you want to get ACL information of.

Note: leading '/' is not necessary, but will still work.

error in (no error)

error in describes error conditions that occur before this node runs. This input provides standard error in functionality.

    status

    status is TRUE (X) if an error occurred before this node ran or FALSE (checkmark) to indicate a warning or that no error occurred before this node ran. The default is FALSE.

    code

    code is the error or warning code. The default is 0.

    source

    source specifies the origin of the error or warning and is, in most cases, the name of the node that produced the error or warning. The default is an empty string.

Version ID

Version ID may be used, if Versioning is enabled on the bucket, and you want to get the ACL information of a specific version of the object. If no Version ID is specified, the latest version of the object will be used.

Default value: empty string - Use the latest version

Region

Region specifies the geographical location of the bucket.

AccessControlPolicy

AccessControlPolicy contains the elements that set the ACL permissions for an object per Grantee.

    Owner

    Owner is the container for the bucket owner's display name and ID.

      ID

      ID is the Canonical User ID of the object owner.

      DisplayName

      DisplayName is the screen name (Email address) of the object owner.

    AccessControlList

    AccessControlList array contains the information about which Grantee has what kind of Permission on the specific Object.

      Grant

      Grant is a container for the grantee and his or her permissions.

        Grantee

        Grantee is the subject whose permissions are being set.

          ID

          ID is the Canonical User ID of the Grantee.

          DisplayName

          DisplayName is the screen name (Email address) of the Grantee.

        Permission

        Permission specifies the permission (FULL_CONTROL, WRITE, WRITE_ACP, READ, READ_ACP) given to the Grantee.

error out

error out passes error or warning information out of a VI to be used by other VIs.

Right-click the error out indicator on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.
