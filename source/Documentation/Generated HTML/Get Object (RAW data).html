LV Interface for Amazon S3.lvlib:Get Object (RAW data).vi

This operation retrieves an object from Amazon S3 and returns RAW data. You must have READ access to the object. If you grant READ access to the anonymous user, you can return the object without using an authorization header. To distribute large files to many people, you can save bandwidth costs using BitTorrent. You can limit returned data using Request parameters.

error in (no error)

error in describes error conditions that occur before this node runs. This input provides standard error in functionality.

    status

    status is TRUE (X) if an error occurred before this node ran or FALSE (checkmark) to indicate a warning or that no error occurred before this node ran. The default is FALSE.

    code

    code is the error or warning code. The default is 0.

    source

    source specifies the origin of the error or warning and is, in most cases, the name of the node that produced the error or warning. The default is an empty string.

Bucket

Bucket specifies the name of the bucket where the object you want to download resides.

Object Path

Object Path is the relative path from the bucket's root to the object you want to get size information of.

Note: leading '/' is not necessary, but will still work.

Request Parameters

Request Parameters allows you to specify conditions that can limit the size of returned data.

    Range

    Range allows you to download the specified range bytes of an object.

      Start

      - If Start is 0, data starting from the object's first byte is returned.

      - If Start is negative, data is returned starting at Start number of bytes before the end of the object.

      - If Start is positive, data is returned starting at Start number of bytes of the object.

      Default value: 0

      Length

      - If Length is 0, the whole object is returned. (default)

      - If Length is negative, all bytes from Start are returned.

      - If Length is positive, Length number of bytes are returned.

      Default value: 0

    If Modified Since

    If Modified Since: Return the object only if it has been modified since the specified time.

    Default value: Not a timestamp - this condition is ignored

    If Unmodified Since

    If Unmodified Since: Return the object only if it has not been modified since the specified time.

    Default value: Not a timestamp - this condition is ignored

    Matching ETag

    Matching ETag: Return the object only if its entity tag (ETag, which is the MD5 sum) is the same as the one specified.

    Default value: empty string - this condition is ignored

    Nonmatching ETag

    Nonmatching ETag: Return the object only if its entity tag (ETag, which is MD5 sum) is different from the one specified.

    Default value: empty string - this condition is ignored

Version ID

Version ID may be used, if Versioning is enabled on the bucket, and you want to download a specific version of the object. If no Version ID is specified, the latest version of the object will be downloaded.

Default value: empty string

Region

Region specifies the geographical location of the bucket.

GetObjectResult

GetObjectResult contains information about the object and the received data.

    Content Length (bytes)

    Content Length (bytes) indicates the size of the returned data in bytes.

    Note: this is not necessarily the same as Object Size

    Content Type

    Content Type indicates the MIME type of the selected object.

    ETag

    ETag indicates the Entity Tag (which is MD5 sum) of the selected object.

    Delete marker

    Delete marker specifies whether the object retrieved was (TRUE) or was not (FALSE) a Delete Marker.

    If true, returned Data is empty.

    Version ID

    Version ID returns the version ID of the retrieved object if it has a unique version ID.

    Object Size (bytes)

    Object Size (bytes) indicates the size of the object referenced by Path in bytes.

    Note: this is not necessarily the same as Content Length

error out

error out passes error or warning information out of a VI to be used by other VIs.

Right-click the error out indicator on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    status

    status is TRUE (X) if an error occurred or FALSE (checkmark) to indicate a warning or that no error occurred.

    Right-click the error out indicator on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    code

    code is the error or warning code.

    Right-click the error out indicator on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    source

    source string describes the origin of the error or warning.

    Right-click the error out indicator on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

Data

Data returns object's received content in its RAW format. You may process, convert (eg. unflatten) or write this data to a file by yourself.
