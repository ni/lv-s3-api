LV Interface for Amazon S3.lvlib:Set Object ACL.vi

This operation sets the access control list (ACL) permissions for an object that already exists in a bucket. (To set the ACL of an object when you put it into a bucket, use Request Parameters' ACL field of the Put Object.vi.) You must have WRITE_ACP permission to set the ACL of an object.

error in (no error)

error in can accept error information wired from VIs previously called. Use this information to decide if any functionality should be bypassed in the event of errors from other VIs.

Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    status

    status is TRUE (X) if an error occurred or FALSE (checkmark) to indicate a warning or that no error occurred.

Object Path

Object Path is the relative path from the bucket's root to the object you want to set ACL information for.

Note: leading '/' is not necessary, but will still work.

Bucket

Bucket specifies the name of the bucket where the object whose ACL information you want to set resides.

Version ID

Version ID may be used, if Versioning is enabled on the bucket, and you want to set the ACL information for a specific version of the object. If no Version ID is specified, the latest version of the object will be used.

Default value: empty string

AccessControlPolicy

AccessControlPolicy contains the elements that set the ACL permissions for an object per Grantee.

    Owner

    Owner is the container for the bucket owner's display name and ID.

      ID

      ID is the Canonical User ID of the Grantee.

      DisplayName

      DisplayName is the screen name (Email address) of the bucket owner.

    AccessControlList

    AccessControlList array contains the information that specify which Grantee has what kind of Permission on the specific Object.

      Grant

      Grant is a container for the grantee and his or her permissions.

        Grantee

        Grantee is the subject whose permissions are being set.

          ID

          ID is the Canonical User ID of the Grantee.

          DisplayName

          DisplayName is the screen name (Email address) of the Grantee.

        Permission

        Permission sets the permission (FULL_CONTROL, WRITE, WRITE_ACP, READ, READ_ACP) given to the Grantee.

Region

Region specifies the geographical location of the bucket.

PutObjectACLResult contains information about the object whose ACL information has been set.

    Version ID

    Version ID shows the Version of the object whose ACL is being set, if versioning support is enabled on the bucket.
