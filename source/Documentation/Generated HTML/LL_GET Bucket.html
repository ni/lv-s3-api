LV Interface for Amazon S3.lvlib:LL_GET Bucket.vi

This core implementation of the GET operation returns some or all (up to 1000) of the objects in a bucket. You can use the request parameters as selection criteria to return a subset of the objects in a bucket. You have direct access to HTTP headers, so you can use this VI to implement special requests or API calls not present in this toolkit.

GET Bucket Request Parameters

GET Bucket Request Parameters allows you to specify special conditions that can limit the amount of returned data.

    delimiter

    delimiter is a character you use to group keys. All keys that contain the same string between the prefix, if specified, and the first occurrence of the delimiter after the prefix are grouped under a single result element, CommonPrefixes. If you don't specify the prefix parameter, then the substring starts at the beginning of the key. The keys that are grouped under CommonPrefixes result element are not returned elsewhere in the response.

    Default value: empty string

    marker

    marker specifies the key to start with when listing objects in a bucket. Amazon S3 lists objects in alphabetical order.

    max-keys

    max-keys sets the maximum number of keys returned in the response body. The response might contain fewer keys but will never contain more.

    If there are additional keys that satisfy the search criteria but were not returned because max-keys was exceeded, IsTruncated? in ListBucketResult is set to TRUE.

    To return the additional keys, see marker.

    If max-keys is less than 1 or greater than 1000, all keys are returned.

    Amazon can not return more than 1000 keys in one request!

    Default value: 0

    prefix

    prefix limits the response to keys that begin with the indicated prefix. You can use prefixes to separate a bucket into different groupings of keys.

    (You can think of using prefix to make groups in the same way you'd use a folder in a file system.)

Headers

Headers array is the list of HTTP Request headers to send with the HTTP Request.

    HTTP Header-Value pair

    HTTP Header-Value pair represents a HTTP header by its name and value.

      Header

      Header specifies the header field to add to the HTTP request. The Header control provides a pull-down menu with some predefined, common header fields.

      Value

      Value specifies the value of the HTTP header.

Bucket

Bucket is the name of the bucket where you want to send the GET request.

error in (no error)

error in describes error conditions that occur before this node runs. This input provides standard error in functionality.

    status

    status is TRUE (X) if an error occurred before this node ran or FALSE (checkmark) to indicate a warning or that no error occurred before this node ran. The default is FALSE.

    code

    code is the error or warning code. The default is 0.

    source

    source specifies the origin of the error or warning and is, in most cases, the name of the node that produced the error or warning. The default is an empty string.

Region

Region specifies the geographical location of the bucket.

ListBucketResult

ListBucketResult contains the bucket contents information returned from Amazon.

    Name

    Name is the name of the bucket.

    Prefix

    Prefix is the same prefix you supplied in GET Bucket Request Parameters.

    Marker

    Marker indicates where in the bucket to begin listing.

    MaxKeys

    MaxKeys is the maximum number of keys returned in the response body.

    IsTruncated?

    IsTruncated? specifies whether (TRUE) or not (FALSE) all of the results were returned. All of the results may not be returned if the number of results exceeds 1000 (maximum number of keys Amazon can return in one request) or that specified by MaxKeys.

    Contents

    Contents array lists information about the objects stored in the specified bucket.

      Content

      Content contains information about an object.

        Key

        Key is the key of the object. Key contains the full path of the object relative to the bucket root.

        Note: Key does not contain the leading "/" that represents the bucket root.

        LastModified

        LastModified shows the Last modification time (in UTC) of the object.

        ETag

        ETag is the Entity Tag (which is an MD5 Sum) of the object. ETag only reflects changes to the contents of an object, not its metadata.

        Size

        Size is the size of the object in bytes.

        Storage Class

        Storage Class specifies the type of storage in terms of redundancy that is used to store the object.

        Possible values are:

        - STANDARD is the default, highly redundant storage type Amazon uses.

        - REDUCED_REDUNDANCY enables customers to reduce their costs by storing non-critical, reproducible data at lower levels of redundancy than Amazon S3's standard storage.

        Owner

        Owner contains elements that identifies the object owner.

          ID

          ID is the Canonical User ID of the object owner.

          DisplayName

          DisplayName is the screen name (Email address) of the object owner.

    CommonPrefixes

    CommonPrefixes array can contain Common Prefixes only if you specify a delimiter.

    When you do, CommonPrefixes contains all (if there are any) keys between Prefix and the next occurrence of the string specified by delimiter.

    In effect, CommonPrefixes lists keys that act like subdirectories in the directory specified by Prefix.

    For example, if prefix is notes/ and delimiter is a slash (/), in notes/summer/july, the common prefix is notes/summer/.

    All of the keys rolled up in a common prefix count as a single return when calculating the number of returns.

      CommonPrefix

      CommonPrefix is a container for the Prefix element.

        Prefix

        Prefix element is a key that begins with the specified Prefix.

error out

error in can accept error information wired from VIs previously called. Use this information to decide if any functionality should be bypassed in the event of errors from other VIs.

Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    status

    status is TRUE (X) if an error occurred or FALSE (checkmark) to indicate a warning or that no error occurred.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    code

    code is the error or warning code.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    source

    source describes the origin of the error or warning.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

Call Result

Call Result returns the result of the call. If the call completed successfully, this output returns "Success", otherwise it will return an error.
