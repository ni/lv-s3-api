LV Interface for Amazon S3.lvlib:List Parts.vi

This operation lists the parts that have been uploaded for a specific multipart upload. This operation must include the upload ID, which you obtain by calling Get Multipart Upload ID.vi.

Upload ID

Upload ID is the Upload's unique identifier string as returned by Amazon's server. You can obtain this ID with the Get Multipart Upload ID.vi from a Multipart Upload RefNum.

Object Path

Path is the path relative from the bucket's root to the object that is being uploaded, and whose already uploaded parts you want to list.

Note: leading '/' is not necessary, but will still work.

Bucket

Bucket specifies the name of the bucket where the object whose already uploaded parts you want to list resides.

Request parameters

    Max parts

    Max parts sets the maximum number of parts to return in ListPartsResult.

    The response might contain fewer parts but will never contain more. If there are additional parts that were not returned because Max parts was exceeded, IsTruncated? in ListPartsResult is set to TRUE.

    To return the additional keys, see Part number marker.

    If Max parts is less than 0, all parts are returned.

    Default value: -1 - Return all parts

    Part number marker

    Part number marker specifies the part after which listing should begin. Only parts with higher part numbers will be listed.

    Note: Part numbers can range from 1 to 10,000.

    Default value: 1

ListPartsResult

ListPartsResult contains information about the upload, the object being uploaded, and the successfully uploaded parts.

    Bucket

    Bucket is the name of the bucket to which the multipart upload was initiated.

    Key

    Key is the object key for which the multipart upload was initiated.

    Upload ID

    Upload ID is the unique ID string returned by Amazon's server identifying the multipart upload whose parts are being listed.

    Initiator

    Initiator contains element that identifies who initiated the multipart upload. If the initiator is an AWS account, this element provides the same information as the Owner element. If the initiator is an IAM User, then this element provides the user ARN and display name.

      ID

      ID provides the Canonical User ID, if the principal is an AWS account. If the principal is an IAM User, it provides a user ARN value.

      DisplayName

      DisplayName is the Principal's screen name (Email address).

    Owner

    Owner contains element that identifies the object owner, after the object is created. If the multipart upload is initiated by an IAM user, this element provides the parent account ID and display name.

      ID

      ID provides the Canonical User ID, if the principal is an AWS account. If the principal is an IAM User, it provides a user ARN value.

      DisplayName

      DisplayName is the Principal's screen name (Email address).

    Storage Class

    Storage Class is the class of storage (STANDARD or REDUCED_REDUNDANCY) used to store the uploaded object.

    Part number marker

    Part number marker is the Part number after which listing begins.

    Next part number marker

    Next part number markeris the number of the upcoming part after last part in the list, when a list is truncated. Use this value as the value to the Part number marker input in a subsequent request to continue listing.

    Max parts

    Max parts is the maximum number of parts that were allowed in the response.

    Is truncated?

    Is truncated? indicates whether the returned list of parts is truncated (TRUE).

    A list can be truncated if the number of parts exceeds the limit returned in the Max parts element.

    Parts

    Parts array lists the parts that have been uploaded. A response can contain zero or more elements.

      Multipart Upload Part

      Multipart Upload Part is a container for elements related to a particular part.

        PartNumber

        PartNumber is the Part number identifying the part.

        Lat modified

        Last modifies is the date and time (in UTC) at which the part was uploaded.

        ETag

        ETag is the Entity tag (which is MD5 sum) returned when the part was uploaded.

        Size

        Size is the size of the uploaded part data in bytes.
