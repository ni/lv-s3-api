LV Interface for Amazon S3.lvlib:Get Bucket Logging.vi

This operation returns the logging status of a bucket and the permissions users have to view and modify that status. To use this operation, you must be the bucket owner.

error in (no error)

error in can accept error information wired from VIs previously called. Use this information to decide if any functionality should be bypassed in the event of errors from other VIs.

Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    status

    status is TRUE (X) if an error occurred or FALSE (checkmark) to indicate a warning or that no error occurred.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

Bucket

Bucket specifies the name of the bucket whose Logging Configuration you want to get.

Region

Region specifies the geographical location of the bucket.

BucketLoggingStatus

BucketLoggingStatus contains information about the Logging configuration of the bucket.

    LoggingEnabled

    LoggingEnabled is TRUE, if Logging is enabled on the bucket. If FALSE, Logging is disabled, thus other fields are empty.

    TargetBucket

    TargetBucket specifies the name of the bucket where server access logs will be delivered.

    TargetPrefix

    TargetPrefix is the prefix for the keys that the log files are being stored under.

    TargetGrants

    TargetGrants array lists granting information entries.

      Grant

      Grant is a container for the grantee and his or her permissions.

        Grantee

        Grantee is the subject whose permissions are being set.

          EmailAddress

          EmailAddress is the Email address of the person whose logging permissions are displayed.

        Permission

        Permission specifies the permission (FULL_CONTROL, WRITE, READ) given to the Grantee.
