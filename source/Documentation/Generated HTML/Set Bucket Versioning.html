LV Interface for Amazon S3.lvlib:Set Bucket Versioning.vi

This operation sets the versioning state of an existing bucket. To set the versioning state, you must be the bucket owner.

Versioning Configuration

Versioning Configuration allows you to specify the Versioning Status and set MfaDelete of the bucket.

    Status

    Status allows you to Enable or Suspend the Versioning capability of the bucket.

    Valid values:

    - Enabled—Enables versioning for the objects in the bucket.

    All objects added to the bucket receive a unique version ID.

    - Suspended—Disables versioning for the objects in the bucket.

    All objects added to the bucket receive the version ID null.

    - Disabled—The same as Suspended.

    This value is allowed only for compatibility purposes, because if the versioning state has never been set on a bucket, it has no versioning state; so a Get Bucket Versioning Configuration request returns Disabled as the versioning Status. Disabled also sets the Versioning to Suspended!

    Default value: Suspended

    MfaDelete

    MfaDelete option allows the bucket owner to restrict some operations only to users who have an Auth device (token) from Amazon associated with the bucket owner's account.

    If the bucket owner enables MFA Delete in the bucket versioning configuration, you must provide the Auth device data.

Auth device data

Auth device data has to be provided to authenticate the request, if Mfa Delete is enabled on the bucket.

    SerialNumber

    Serial number is the serial number of your Authentication device (token).

    TokenCode

    TokenCode is the value currently displayed on your Authentication device (token).

Bucket

Bucket specifies the name of the bucket whose Versioning Configuration you want to set.

Region

Region specifies the geographical location of the bucket.
