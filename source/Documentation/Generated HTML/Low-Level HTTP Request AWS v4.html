LV Interface for Amazon S3.lvlib:Low-Level HTTP Request AWS v4.vi

This core implementation of AWS v4 HTTP requests may be used to implement special requests or API calls not present in this toolkit.

error in (no error)

error in can accept error information wired from VIs previously called. Use this information to decide if any functionality should be bypassed in the event of errors from other VIs.

Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    status

    status is TRUE (X) if an error occurred or FALSE (checkmark) to indicate a warning or that no error occurred.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    code

    code is the error or warning code.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    source

    source describes the origin of the error or warning.

    Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

    request body

    request body is the body that is sent with the HTTP request.

HTTP-Verb

HTTP-Verb is the RESTful method that will be invoked.

AWS Service

AWS Service is the service you want to make API calls to.

If you decide to use this VI do other AWS service, make sure 'Bucket' and 'Object Key' parameters are left empty.

Region

Region specifies the geographical location of the bucket.

Bucket

Bucket specifies the name of the bucket you want to create/delete or interact with.

Object Key

Object Key is the path to an object in a bucket.

Note: leading '/' is not necessary, but will still work.
