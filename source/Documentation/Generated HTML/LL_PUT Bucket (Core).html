LV Interface for Amazon S3.lvlib:LL_PUT Bucket (Core).vi

Note: This VI is no longer supported. Use the Low-Level HTTP Request AWS v4 VI instead.

This core implementation of the PUT operation may be used to implement special requests or API calls not present in this toolkit.

Data

Data is the RAW data you want to send as the data part of the PUT request.

Headers

Headers array is the list of HTTP Request headers to send with the HTTP Request.

    HTTP Header-Value pair

    HTTP Header-Value pair represents a HTTP header by its name and value.

      Header

      Header specifies the header field to add to the HTTP request. The Header control provides a pull-down menu with some predefined, common header fields.

      Value

      Value specifies the value of the HTTP header.

Path and query string

Path and query string is the path and query part of the URI where the request is going to be sent. This string has to start with a "/" which represents the root of the bucket's URI.

Default value: /

Bucket

Bucket is the name of the bucket where you want to send the PUT request.

error in (no error)

error in describes error conditions that occur before this node runs. This input provides standard error in functionality.

    status

    status is TRUE (X) if an error occurred before this node ran or FALSE (checkmark) to indicate a warning or that no error occurred before this node ran. The default is FALSE.

    code

    code is the error or warning code. The default is 0.

    source

    source specifies the origin of the error or warning and is, in most cases, the name of the node that produced the error or warning. The default is an empty string.

Response header

Response header is the header part of the HTTP response.

Response body

Response body is the body part of the HTTP response.

Call Result

Call Result returns the result of the call. If the call completed successfully, this output returns "Success", otherwise it will return an error.

error out

error out passes error or warning information out of a VI to be used by other VIs.

Right-click the error out indicator on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.
