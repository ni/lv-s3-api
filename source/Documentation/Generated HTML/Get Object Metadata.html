LV Interface for Amazon S3.lvlib:Get Object Metadata.vi

This operation retrieves information and metadata associated with an object from Amazon S3. You must have READ access to the object. If you grant READ access to the anonymous user, you can return the object without using an authorization header.

error in (no error)

error in describes error conditions that occur before this node runs. This input provides standard error in functionality.

    status

    status is TRUE (X) if an error occurred before this node ran or FALSE (checkmark) to indicate a warning or that no error occurred before this node ran. The default is FALSE.

    code

    code is the error or warning code. The default is 0.

    source

    source specifies the origin of the error or warning and is, in most cases, the name of the node that produced the error or warning. The default is an empty string.

Bucket

Bucket specifies the name of the bucket where the object whose metadata you want to get resides.

Object Path

Object Path is the path to the object whose metadata you want to get, relative from the bucket's root.

Note: leading '/' is not necessary, but will still work.

Request Parameters

Request Parameters allows you to specify conditions that can limit the size of returned data.

    Range

    Range container is unused.

      Start

      Start is unused. Any value wired to this input has absolutely no effect on the functioning of this VI.

      Length

      Length is unused. Any value wired to this input has absolutely no effect on the functioning of this VI.

    If Modified Since

    If Modified Since: Return the object's metadata only if it has been modified since the specified time.

    Default value: Not a timestamp - this condition is ignored

    If Unmodified Since

    If Unmodified Since: Return the object's metadata only if it has not been modified since the specified time.

    Default value: Not a timestamp - this condition is ignored

    Matching ETag

    Matching ETag: Return the object's metadata only if its entity tag (ETag, which is the MD5 sum) is the same as the one specified.

    Default value: empty string - this condition is ignored

    Nonmatching ETag

    Nonmatching ETag: Return the object's metadata only if its entity tag (ETag, which is MD5 sum) is different from the one specified.

    Default value: empty string - this condition is ignored

Version ID

Version ID may be used, if Versioning is enabled on the bucket, and you want to get metadata of a specific version of the object. If no Version ID is specified, the latest version of the object will be downloaded.

Default value: empty string

Region

Region specifies the geographical location of the bucket.

HeadObjectResult

HeadObjectResult contains information about the object and the object's metadata.

    Object data

    Object data is a container that shows information about the object itself.

      Content Length (bytes)

      Content Length (bytes) is meaningless, so DO NOT USE!

      Content Type

      Content Type indicates the MIME type of the selected object.

      ETag

      ETag indicates the Entity Tag (which is MD5 sum) of the selected object.

      Delete marker

      Delete marker specifies whether the object whose metadata retrieved was (TRUE) or was not (FALSE) a Delete Marker.

      Version ID

      Version ID returns the version ID of the object whose metadata was retreived. if it has a unique version ID.

      Object Size (bytes)

      Object Size (bytes) indicates the size of the object referenced by Path in bytes.

    Metadata

    Metadata array contains all the metadata (key-value pairs) associated with the object.

      Key-Value pair

      Key-Value pair is a container for a key and it's value.

        Key

        Key is the name of the metadata whose value is set in the Value field.

        Value

        Value is the value of the actual metadata referenced by Key.
