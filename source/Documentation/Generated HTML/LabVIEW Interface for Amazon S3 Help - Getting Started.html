LabVIEW Interface for Amazon S3

Installed With: VI Package Manager

Getting Started with LabVIEW Interface for Amazon S3

  1. Registering for Amazon S3

    2. The first step to use this toolkit is to register an AWS account at the Amazon Web Services portal.

    After registration has been completed, you will be notified by e-mail.

  2. AWS Account Security Credentials

    3. When you create an AWS Account, AWS assigns the following credentials to you:

    • Access Key ID (a 20-character, alphanumeric string). For example: AKIAIOSFODNN7EXAMPLE
    • Secret Access Key (a 40-character string). For example: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
    The Access Key ID uniquely identifies an AWS Account. You can use these AWS Account credentials to send authenticated requests to Amazon S3. You will need these Security Credentials while using this toolkit.
    Your Access Key ID and Secret Access Key are displayed when you create your AWS account. They are not e-mailed to you. If you need to see them again, you can view them at any time from your AWS account.
    • Go to the Amazon Web Services web site at http://aws.amazon.com.
    • Click the Account tab.
    • Click Security Credentials.
    If you have not already signed in, Sign In or Create an AWS Account page displays. You must sign in before you can see your AWS account security credentials. Your access credentials, Access Key ID and Secret Access Key, are displayed on the resulting Security Credentials page.

  3. Using toolkit VIs

    4. Using Examples

    This toolkit comes with some example applications that demonstrate the usage of the API elements.
    You can locate these examples using NI Example Finder (located under Help -> Find Examples...) by searching for the keyword "s3" or browsing the Examples tree (Networking -> Internet & Web).

    Creating you own Application

    You can find the API functions on the "Functions -> Addons -> LabVIEW Interface for Amazon S3" palette.

    To start using S3, first you have to store your Security credentials in the toolkit's global credential store within your application. Use the "Set AWS Credentials.vi" to set and store your credentials. (For security purposes there is no way to read out the credentials from the store.) You only need to store the credentials once, before calling any S3 API calls.

    After storing your credentials, you can use other S3 functions:

    • You can list the buckets owned by you using the "List Buckets" function.
    • Operations with/on buckets are found under the "Bucket Operations" subpalette.
    • Operations with/on objects are found under the "Object Operations" subpalette.
    • If you are working with large objects, you can use the Multipart upload functions under the "Object Operations -> Advanced Object Operations -> Multipart Upload" subpalette. These functions provide you with the ability to upload your data in smaller chunks.
    • Functions on the "Low-Level" subpalette allows you to implement Amazon S3 API calls that are not implemented in this toolkit. Thes VIs are also used by the higher level functions of this toolkit.
