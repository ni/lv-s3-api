LV Interface for Amazon S3.lvlib:Put Object from RAW data.vi

This operation adds an object to a bucket. You must have WRITE permissions on a bucket to add an object to it. Amazon S3 never adds partial objects; if you receive no error response, Amazon S3 added the entire object to the bucket. Amazon S3 is a distributed system. If Amazon S3 receives multiple write requests for the same object simultaneously, all but the last object written will be overwritten. Amazon S3 does not provide object locking; if you need this, make sure to build it into your application layer or use versioning instead.

Request Parameters

Request Parameters allows to specify some advanced features of S3 to use to store the object.

    Content Type

    Content Type is the standard MIME type describing the format of the contents. For more information, go to http://www.w3.org/Protocols/rfc2616/rfc2616-sec14.html#sec14.17.

    Default value: binary/octet-stream

    ACL

    ACL is the canned ACL to apply to the object. You can limit access of other people to the object by setting it's ACL.

    Default value: private - Only it's owner can access the object

    Storage Class

    Storage Class allows you to specify the type of storage in terms of redundancy that will be used to store data.

    Possible values are:

    - STANDARD is the default, highly redundant storage type Amazon uses.

    - REDUCED_REDUNDANCY enables customers to reduce their costs by storing non-critical, reproducible data at lower levels of redundancy than Amazon S3's standard storage.

    User Metadata

    User Metadata array allows you to store some metadata associated with the object. The total size of the HTTP request that includes the User Metadata - not including the body ( Data)-, must be less than 8 KB.

      Object Metadata

      Object Metadata represents a metadata associated with an object in the form of a key-value pair.

        Key

        Key specifies the name of the metadata you want to store with the object.

        Value

        Value specifies the value of the actual metadata referenced by Key you want to store with the object.

    Content MD5

    Content MD5 is the base64 encoded 128-bit MD5 digest of the Data according to RFC 1864. This information can be used as

    a message integrity check to verify that the data is the same data

    that was originally sent. Although it is optional, Amazon recommends using the Content-MD5 mechanism as an end-to-end integrity check.

    Default value: empty string - MD5 is not used

Object Path

Object Path is the relative path from the bucket's root to the object you want to get size information of.

Note: leading '/' is not necessary, but will still work.

Bucket

Bucket specifies the name of the bucket you want to use to store the object.

Data

Data is the RAW data that will be stored as the content of the object.

Region

Region specifies the geographical location of the bucket.

PutObjectResult

PutObjectResult contains information about the object if it has been successfully created.

    ETag

    ETag contains the Entity Tag (which is MD5 sum) of the stored object.

    Version ID

    Version ID contains the Version ID of the stored object if Versioning is enabled on the bucket.
